Diseño de Software

Principios fundamentales, conceptos clave y mejores prácticas

¿Qué es el Diseño de Software?

Proceso fundamental en la ingeniería de software que busca definir la estructura de un sistema informático mediante la conceptualización, implementación, puesta en funcionamiento y modificación de sistemas complejos.

Definición IEEE

Según IEEE Std 1471-2000, la arquitectura de software es la organización fundamental de un sistema enmarcada en sus componentes, las relaciones entre ellos, el ambiente, y los principios que orientan su diseño y evolución.

Propósitos del Diseño

  • Descomponer un sistema en elementos de software
  • Asignar y describir funciones para cada elemento
  • Establecer relaciones e interacciones entre componentes
  • Satisfacer requisitos funcionales y no funcionales

Principios SOLID

Single Responsibility

Una clase debe tener una única responsabilidad y una sola razón para cambiar.

Open-Closed

Las clases deben estar abiertas a la extensión pero cerradas a su modificación.

Liskov Substitution

Las subclases deben ser sustituibles por sus clases base sin producir resultados extraños.

Interface Segregation

Muchas interfaces específicas son mejores que una interfaz de propósito general.

Dependency Inversion

Las clases deben depender de interfaces o clases abstractas en lugar de clases concretas.
